# Command Writing Rules

`.claude/commands/` 파일은 `/명령어` 입력 시 Claude에게 그대로 전달되는 프롬프트다.
"설명 문서"가 아니라 "실행 지시문"으로 작성해야 작동한다.

## 형식 규칙

- 파일 첫 줄은 "~하라" 형식의 직접 지시문으로 시작한다
- `## 사용법`, `## 실행 내용` 같은 메타 섹션 금지 — 실행 단계만 작성
- 단계는 `## 1단계:`, `## 2단계:` 형식으로 번호를 붙인다
- 마지막에 "## 완료 후" 섹션으로 출력 형식을 명시한다

## $ARGUMENTS 규칙

- 인자가 필요한 명령은 파일 첫 단락에 `$ARGUMENTS`가 비었을 때 처리를 명시한다
- 예: `$ARGUMENTS가 비어 있으면 "사용법: /명령어 <인자>" 출력 후 중단한다`
- `$ARGUMENTS`는 파일 내 여러 곳에 사용 가능 (경로, 클래스명 등)

## 경로 규칙

- 모든 파일 경로는 프로젝트 루트 기준 상대경로로 작성한다
- 옳음: `extension/games/$ARGUMENTS.js`
- 틀림: `/extension/games/$ARGUMENTS.js` (앞에 `/` 붙이면 절대경로로 해석될 수 있음)

## 디렉토리 생성

- 파일을 생성하는 명령은 반드시 대상 디렉토리를 먼저 생성하는 단계를 포함한다
- Bash `mkdir -p`로 생성하며, 이미 존재해도 오류 없이 진행한다

## 출력 규칙

- 완료 후 출력은 한 줄 요약 또는 생성 파일 목록으로 제한한다
- 불필요한 설명, 재확인 질문 금지
